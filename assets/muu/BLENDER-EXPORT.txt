## テクスチャが黒い／欠ける — 原因（Pack だけでは直らない）

`File → External Data → Pack Resources` は **画像ファイルを .blend に埋め込む** だけ。
glTF/GLB export は **Principled BSDF に直接つながった Image Texture だけ** 書き出す。

HumGen の `.Human` マテリアルは **カスタムノードグループ** で肌色を合成している。
Pack しても export 時に Principled の Base Color へ届かないテクスチャは GLB に入らない。

### speak-mou.glb を調査した結果（2026-07）

| マテリアル | GLB 内 | 問題 |
|-----------|--------|------|
| `.Human`（体・手足） | baseColor = **そばかす用** `.fm_0_freckles_body` のみ | **肌の Diffuse が未 export** → 手足が黒く見える |
| `Brain Meat.001` | テクスチャなし | Geometry Nodes / 手続きシェーダー → GLB 非対応 |
| `.HG_Eyes_Outer_FAST` | テクスチャなし | 虹彩の外側レイヤーが export されない |
| `.HG_Teeth` / `.HG_Eyes_Inner` | ✓ 正常 | — |

Pack は成功している（GLB 内に 8 枚 embed 済み）。**接続の問題** で Pack では解決しない。

---

## 正しい直し方 — HumGen Texture Baking（必須）

1. キャラ（speak-mou）を選択
2. HumGen パネル → **Process** タブ
3. **Texture Baking** を有効化
   - Sample count: 128 以上推奨
   - Body / Face など Resolution を 2K〜4K に
4. 下部の出力フォルダ（空なら HG content packs の `output`）
5. **Bake** 実行 → Principled BSDF 向けに flattened テクスチャが生成される
6. 保存してから GLB export

参考: https://help.humgen3d.com/baking

Process タブが見えない場合（ARP リマップ後など）:
- HumGen パネルでキャラを Human として再選択
- または Shading で `.Human` を開き、Base Color を Image Texture 1 枚に手動で繋ぎ直してから export

---

## Brain（脳）のテクスチャ

Brain は Geometry Nodes だと GLB に載らないことが多い。

**Bake 手順（Brain.001 選択）:**
1. Shading タブで見た目を確認
2. Render Engine: Cycles（Eevee でも可）
3. メッシュ選択 → Render → **Bake** → Type: Diffuse（Direct+Indirect オフ、Color のみ）
4. Image Texture ノードを追加して New Image → **Bake**
5. Image → Save As PNG
6. その PNG を Principled BSDF の Base Color に繋ぎ直して export
